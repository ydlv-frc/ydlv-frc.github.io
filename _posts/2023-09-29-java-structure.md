בדרך כלל, הדבר הראשון שעושים כשלומדים שפת תכנות הוא ללמוד איך לכתוב תוכנית שכותבת למסך את המילים "Hello, world!" - הדבר כל כך נפוץ שיש לזה שם, זה נקרא `תוכנית "שלום עולם"`. למה להתחיל מלכתוב למסך? כי לרוב זה הקל ביותר; למה "Hello, world!"? כנראה סיבות היסטוריות כלשהן. בכל מקרה, כך נראה קוד "שלום עולם" בפייתון (כן, פייתון - אנחנו עובדים בשפת Java אבל יש סיבה שאני מציג קודם את פייתון):

```py
print("Hello, world!")
```

פשוט - פקודה אחת, שורה אחת. ב-Java הפקודה המתאימה תיראה כך:

```java
System.out.println("Hello, world");
```

אוקיי, קצת יותר טקסט, אבל לא מאוד שונה. זו **הפקודה** המתאימה לכך, וגם כאן, זו רק פקודה אחת, והיא תופסת רק שורה אחת. אבל ב-Java, אפילו בשביל התוכנית הפשוטה הזו, צריך שלד מסביבה. בשביל פייתון, קובץ שמכיל רק את השורה שכתבנו למעלה ידפיס את הפלט שאנחנו רוצים; אבל ב-Java, הפקודה שאנחנו צריכים להריץ צריכה להיות בתוך מסגרת מוזרה סביבה:

```java
package learning_java.hello_world;

public class FirstCode {
    public static void main(String[] args) {
        System.out.println("Hello, world!");
    }
}
```

בקוד הזה עדיין יש רק פקודה אחת שמדפיסה "Hello, world!" והיא תופסת רק שורה אחת, זו שבאמצע הקוד, אבל השלד מסביבה הכרחי. כדי להבין את הפקודה שבתכנית הראשית, אקדיש את הפוסט הבא. **בפוסט הזה, אני רוצה להסביר, רק קצת, את מה שמסביב. רק ברמה שתספיק לכן להריץ קוד Java, לפתור שגיאות שקשורות בעניינים האלה, ובתקווה מעט להרגיע את הפחד שהדבר המוזר הזה מסביב אולי יוצר. <u>אפשר לדלג על הפוסט הזה</u> ולחזור אליו אם נתקלים בבעיות בהרצה של קוד Java על המחשב שלכן**. 

לפוסט הזה אני אניח שהתקנתן את Java על המחשב ואת Visual Studio Code WPI Edition. אני לא אפרסם כאן מדריך טכני של מה ואיך להתקין - באתר של WPI ניתן למצוא הוראות של מה צריך להתקין, ובנוסף של מה להתקין שם אני ממליץ להתקין את הJDK [מכאן](https://www.oracle.com/il-en/java/technologies/downloads/). על Visual Studio Code עצמו אני ממליץ לוודא שמותקן [חבילת ההרחבות של Microsoft ל-Java](https://code.visualstudio.com/docs/java/extensions).

# מה הולך כאן?

## טרמינל

נתחיל בלהסביר את הדבר המוזר הזה כאן למטה. פעם, לפני שהיו במחשבים תוכנות גרפיות עם חלונות שאפשר להזיז ועכבר ששולט בהן, מסך המחשב היה טרמינל - שורת פקודה: היית כותב למחשב פקודה, הוא היה מבצע אותה ומציג פלט, וחוזר חלילה. מעין צ'אט עם המחשב. וזה עדיין קיים גם במחשב שלכן. זה מה ש-Visual Studio Code עשה כשלחצתן על Run: פתח שורת פקודה, כתב פקודה עבורה, שלח אותה, והמחשב ביצע אותה. Visual Studio Code מציג לנו את הפקודה שהוא ביצע, וזה הטקסט הכחול המוזר הזה שמתחיל ב-`&` וכל כך ארוך שהוא לוקח יותר משורה אחת, אבל כל הטקסט הזה לא משנה, כי Visual Studio Code עושה את זה בשבילנו. זה אחד מהיתרונות הרבים של עורכי קוד מודרניים - להיות מתכנתת או מתכנת היום זה יותר קל ממה שזה היה פעם. אבל מה עושה התוכנה?

## קומפילציה והרצה

מה שהמחשב שלנו עשה מורכב משני חלקים: ראשית, הקומפיילר של Java לקח את קוד ה-Java שלנו, וקימפל אותו - הפך אותו לקבצים בינאריים שאותם Java יכולה להריץ. לאחר מכן, Java לקחה את הקבצים הבינאריים האלה, והריצה אותם. נכנסתי לפרטים [בפוסט העשרה קודם]({{ site.url }}{{ site.baseurl }}/2023/09/28/what-is-java/), אז לא אעשה זאת שוב כאן. אם תבדקו את תיקיית bin אצלכן בפרויקט, תמצאו שיש שם תיקיה בשם `learning_java` ובתוכה תיקיה בשם `hello_world`; ובתוכה קובץ בשם `FirstCode.class`. קבצי `.class` הם הקבצים הבינאריים של Java. הם לא יהיו קריאים מאוד לבני אדם, והם לא צריכים להיות - תוכנה קוראת אותם ומבצעת אותם.

## חבילות ושמותיהן

קוד ב-Java מאורגן בחבילות. חבילות הן תיקיות שבהן קבצי קוד ה-Java שלנו נמצאים, וכיוון שמדובר בתיקיות, הן יכולות להיות מאורגנות במבנה היררכי: אחת בתוך השניה. תיקיית הקוד שלנו היא `src`, וזו הנקודה בה "מתחיל" מבנה החבילות שלנו. תיקיה כזו נקראת באנגלית גם `sources root` ("שורש קבצי המקור") - המילה `sources` מתייחסת לקבצי מקור, שזה שם מהולל לקבצי הקוד שאנחנו כותבות, ו-`root` הוא מילה ממדעי המחשב שמשמעותה "מבנה היררכי מתחיל כאן".

{: .notice--info}

אנשי מדעי המחשב קוראים למבנים היררכיים כאלה בשם "עצים", למרות שהם מציירים אותם כמו עצים **הפוכים** שצומחים מלמעלה למטה (יש מי שאומרים שזה בגלל שהם לא ראו עץ אמיתי מעולם שכן הם לא יוצאים מהבית), ובכל הטרמינולוגיה הזו יש גם דברים שנקראים צמתים, ולפעמים גם דברים בשם קשתות, ולפעמים לקשתות יש משקל... לאנשי מדעי המחשב יש שמות מוזרים לדברים לפעמים.

את הקוד שלנו שמנו בחבילה בשם `learning_java.hello_world`, שזו בעצם תיקיה בשם `hello_world` בתוך תיקיה בשם `learning_java` שיושבת בתוך תיקיית `src` שלנו. זו דוגמה למבנה ההיררכי שהזכרתי. שימו לב שהתיקיות מופרדות בנקודה.

קובץ Java צריך להתחיל בפקודת `package` שאומרת באיזו חבילה הוא נמצא. הפקודה צריכה להכיל את החבילה **הנכונה,** בהתאם למיקום של הקובץ. אם הם לא מתאימים, זוהי **שגיאת קומפילציה:** הקוד, מבחינת Java, אינו הגיוני. זה מה שקרה קודם, כאשר כתבנו את פקודת ה-`package` בקובץ שלנו, ובהתאם לכך, Visual Studio Code זיהה את השגיאה והציע לנו שתי דרכים לתקן אותה: להזיז את הקובץ לתיקיה המתאימה (באופציה הזו בחרנו), או לשנות את פקודת ה-`package`.

בנוסף, פקודת `package` צריכה להסתיים בנקודה-פסיק (`;`). למעשה, כל פקודה ב-Java צריכה להסתיים בנקודה-פסיק.

יש כללים ב-Windows ובמערכות הפעלה אחרות לשמות של תיקיות; לא הכל יכול להיות שם של תיקיה, למשל אסורים סימני פיסוק מסוימים (לא כולם); ב-Windows יש גם שמות מסוימים שאסורים מסיבות אחרות - אי אפשר לקרוא לתיקיה con (כן, ברצינות, תנסו - זה לא רלוונטי לנו כאן, אבל מי שמתעניינת למה, לגמרי כהעשרה, יכולה לצפות בסרטון הזה [כאן](https://www.youtube.com/watch?v=bC6tngl0PTI&pp=ygUOdG9tIHNjb3R0IGNvbiA%3D) - אגב, אני ממליץ על ערוץ היוטיוב הזה בכללי, הוא עושה תכנים מצוינים בכל מיני תחומים).

אז אם משהו לא יכול להיות שם של תיקיה במערכת ההפעלה שלנו, הוא לא יכול להיות שם של חבילה אצלנו, **אבל ב-Java יש כללים מגבילים יותר**. 

ב-Java, יש מה שנקרא idetifiers - אפשר לתרגם את זה כ"מזהה", אבל אני מעדיף לתרגם את זה כ"שם". המילה הזו מתייחסת לשם באופן כללי ב-Java, כולל שמות של חבילות, וגם של אלמנטים אחרים בשפה - בגדול, כל דבר שהמתכנתת נותנת לה שם. אנחנו עוד לא מכירים את האלמנטים, אבל הנה רשימת הכללים:

הכללים האלה **מחייבים** במובן של הקומפיילר מחייב אותן - קוד ב-Java שלא עומד בכללים האלה אינו הגיוני מבחינת הקומפיילר ו-Java אינה יכולה לקמפל אותו, כלומר להפוך אותו לקוד בר-הרצה.

בנוסף, **מקובל** שכל האותיות בשמות של חבילות הן אותיות קטנות, אבל זה לא משהו שהקומפיילר של Java מחייב אותנו; זה כלל שמקובל על רוב מתכנתי Java, או, מה שנקרא, **קונבנציה, או בעברית, מוסכמה.** אני מרכז רשימה של כמה מוסכמות חשובות [כאן]({{ site.url }}{{ site.baseurl }}/conventions/).

לגבי מה עושים עם שם החבילה שלנו צריך לכלול יותר ממילה אחת, אין הסכמה אוניברסלית - יש מי שחושבים שלא צריך להפריד בין המילים, ויש מי שחושבים שצריך להפריד בין המילים עם קו תחתון. כמו שאולי שמתן לב, אני הולך על השניה מביניהן: `learning_java.hello_world`. סגנון זה נקרא lower_snake_case (המילה case מתייחסת לסוג אות באנגלית - קטנות או גדולות, המילה lower מתייחסת לאותיות קטנות, ואני מניח שאפשר לראות למה snake).

מילה אחרונה לגבי חבילות: ניתן למקם קבצי Java ישר בתיקייה הראשית של קבצי ה-Java (מה שקראתי לו קודם "sources root"). זה נקרא `default package` (חבילת ברירת המחדל) ובמקרה זה לא יהיה בקובץ פקודת `package`. **מומלץ מאוד להמנע מלהשתמש באופציה הזו למרות שהיא טכנית מותרת. היא עושה בעיות. פשוט נשתמש בחבילות.**

## מחלקות, פונקציות ושמותיהן

קובץ ב-Java, מכיל, באופן כללי, מחלקה אחת בלבד. מהי מחלקה? זה לא מאוד משנה עכשיו: מה שמשנה הוא שהיא מתחילה במילים `public class` ואחריה שם המחלקה, ולאחר מכן פותחים סוגריים מסולסלים. כל מה שנמצא בתוך הסוגריים המסולסלים הוא חלק מהמחלקה. ב-Java, השאלה איפה דברים מתחילים ואיפה הם נגמרים נענית על ידי סוגריים מסולסלים.

{% capture python_scope %}

בפייתון התשובה היא "מתחיל בנקודותיים ונמשך לפי ריווח", כלומר זה אחד ההבדלים בין Java לפייתון. למשל, מי שמכירה if בפייתון, הסתכלי בקוד הבא:

```python
print("Hey. ")
if temperature > 30:
    print("It sure is hot today!")
print("How are you?")
```

פייתון יודעת מה בתוך התנאי לפי הריווח. הקוד המתאים ב-Java הוא:

```java
System.out.println("Hey.");
if(temperature > 30) {
    System.out.println("It sure is hot today!");
}
System.out.println("How are you?");
```

שפת Java יודעת מה בתוך התנאי לפי הסוגריים המסולסלים. גם כאן שמנו ריווח, כי כך **נהוג** ב-Java וזה יותר נוח בעיניים וברור לקריאה, אבל ב-Java, הריווח הוא בשבילנו - הקומפיילר לא מתייחס אליו, ומבחינת Java, היינו גם יכולים לשים את כל הקוד בשורה אחת (Java מפרידה פקודות על ידי נקודה-פסיק, לא על ידי ירידת שורה, כך שגם ירידות שורה הן רק בשבילנו).

שם המחלקה צריך לעמוד בכללים שנכתבו למעלה, ובנוסף, שם קובץ ה-Java שבו הוא נמצא צריך להתאים. כך למשל הקוד שלנו נמצא בקובץ בשם `FirstCode.java` ויש בו מחלקה בשם `Firstcode`. בנוסף, גם לגבי שם מחלקה יש מוסכמות: המוסכמה היא ששם של מחלקה צריך להתחיל באות גדולה, ואם יש יותר ממילה אחת, אז גם כל מילה נוספת צריכה להתחיל באות גדולה. זה נקרא PascalCase או UpperCamelCase.

כרגע מה שיש במחלקה שלנו הוא פונקציה בשם `main`. זהו הדבר שאותו אנחנו "פותחים" בשורה `public static void...`, ותוכן הפונקציה הוא מה שנמצא בתוך הסוגריים המסולסלים שמתחילים בשורה הזו. מה זו פונקציה? כרגע גם זה לא חשוב מאוד; לבינתיים, השאלה מהן פונקציות לא מעניינת אותנו בכלל, ומי שצריכה תשובה כלשהי, אז לבינתיים, פונקציה היא רצף של פקודות שעוטפים אותו עם שם.

אצלנו בקוד בינתיים זו תהיה הפונקציה היחידה, כלומר, כפי שאמרתי בהתחלה, נכתוב רק בתוך הסוגריים המסולסלים של ה-main, עד שנגיע לחלק בו אנחנו לומדים מהן פונקציות. עכשיו אפשר לקצר ולהגיד למה היינו צריכים את הכל, שורת מחץ אחת:

**קוד ב-Java מתחיל לרוץ מתוך פונקציית ה-main, שצריכה להיות בתוך מחלקה, שצריכה להיות בתוך קובץ עם שם מתאים, ולהימצא בחבילה כלשהי.**
